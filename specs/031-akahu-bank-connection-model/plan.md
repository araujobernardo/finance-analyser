# Implementation Plan: Akahu Bank Connection Data Model

**Branch**: `771-akahu-bank-connection-model` | **Date**: 2026-05-31 | **Spec**: [spec.md](spec.md)
**Feature ID**: FA-BANK-001

## Summary

Introduce the persistence layer for Akahu bank connectivity: two new database
tables (`akahu_connections` and `akahu_account_links`), an AES-256-GCM
encryption utility for the Akahu user token, and the corresponding Drizzle
schema definitions and TypeScript types. No API endpoints, UI, or sync logic
are included — those are FA-BANK-002 and FA-BANK-003.

## Technical Context

**Language/Version**: TypeScript (Node.js 20, ESM)
**Primary Dependencies**: Drizzle ORM (`drizzle-orm/pg-core`), Node.js built-in `crypto` (no new npm packages required)
**Storage**: PostgreSQL — Drizzle migration `0009_akahu_bank_integration.sql`
**Testing**: Vitest — unit tests for `encryption.ts`; no DB integration tests in this story
**Target Platform**: Render Web Service (Express / Node.js server)
**Project Type**: Web application — backend data layer only
**Performance Goals**: N/A — data model and utility only
**Constraints**: Akahu user token MUST NOT be stored in plaintext at any point. `ENCRYPTION_KEY` env var must be present at runtime or the utility throws immediately.
**Scale/Scope**: Single-user app; data model is scoped per user as a forward-compatibility safeguard.
