## Implementation Phases

### Phase 1 — Install Playwright

**T001** Install `@playwright/test` as a dev dependency:

```bash
npm install --save-dev @playwright/test
```

**T002** Create `playwright.config.ts` at the project root:

```typescript
import { defineConfig, devices } from "@playwright/test";
import "dotenv/config";

export default defineConfig({
  testDir: "./e2e",
  timeout: 30_000,
  retries: process.env.CI ? 1 : 0,
  use: {
    baseURL: process.env.E2E_BASE_URL ?? "http://localhost:5173",
    headless: !!process.env.CI,
  },
  projects: [
    {
      name: "chromium",
      use: { ...devices["Desktop Chrome"] },
    },
  ],
});
```

### Phase 2 — Write the E2E test

**T003** Create `e2e/auth.spec.ts`:

```typescript
import { test, expect } from "@playwright/test";

const email = process.env.E2E_EMAIL!;
const password = process.env.E2E_PASSWORD!;

test("sign-in and account load", async ({ page }) => {
  await page.goto("/login");

  await page.getByLabel(/email/i).fill(email);
  await page.getByLabel(/password/i).fill(password);
  await page.getByRole("button", { name: /sign in/i }).click();

  await page.waitForURL((url) => !url.pathname.includes("/login"), {
    timeout: 15_000,
  });

  await expect(page.getByText("Failed to load accounts")).not.toBeVisible();

  const accountItems = page.locator("[data-testid='account-item']");
  await expect(accountItems.first()).toBeVisible({ timeout: 15_000 });
});
```

> **Note on selectors**: The `[data-testid='account-item']` selector may need adjusting to match the actual DOM. Fallback: `page.locator(".sidebar").locator("li").first()`. Check the sidebar component to confirm the correct selector before the PR is merged.

### Phase 3 — Scripts and environment

**T004** Add `e2e` script to `package.json`:

```json
"e2e": "playwright test"
```

**T005** Add to `.env.example`:

```
E2E_BASE_URL=http://localhost:5173
E2E_EMAIL=your-test-user@example.com
E2E_PASSWORD=your-test-password
```

### Phase 4 — CI integration

**T006** Add `e2e` job to `.github/workflows/ci.yml` after the existing `quality` job:

```yaml
e2e:
  name: E2E Tests
  runs-on: ubuntu-latest
  needs: [quality]
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: 20
        cache: "npm"
    - run: npm ci
    - name: Install Playwright browsers
      run: npx playwright install --with-deps chromium
    - name: Run E2E tests
      run: npm run e2e
      env:
        E2E_BASE_URL: https://finance-analyser-seven.vercel.app
        E2E_EMAIL: ${{ secrets.E2E_EMAIL }}
        E2E_PASSWORD: ${{ secrets.E2E_PASSWORD }}
    - name: Upload Playwright report
      if: failure()
      uses: actions/upload-artifact@v4
      with:
        name: playwright-report
        path: playwright-report/
        retention-days: 7
```

## Key Decisions

| Decision    | Choice                     | Rationale                                          |
| ----------- | -------------------------- | -------------------------------------------------- |
| Test runner | `@playwright/test`         | Built-in runner, no Vitest integration needed      |
| CI target   | Live production Vercel URL | Avoids complex service orchestration in CI         |
| Browser     | Chromium only              | Minimal scope per spec; multi-browser out of scope |
| Headless    | CI=headless, local=headed  | Easier local debugging                             |
| Retries     | 1 in CI, 0 locally         | Handles transient flakes, doesn't mask failures    |
| Credentials | GitHub Secrets + `.env`    | Never hardcoded; matches existing project pattern  |

## GitHub Actions Secrets Required

Add these in repo **Settings → Secrets and variables → Actions**:

- `E2E_EMAIL` — test user's email address
- `E2E_PASSWORD` — test user's password

`E2E_BASE_URL` is hardcoded in the CI job to the production URL and is not a secret.
