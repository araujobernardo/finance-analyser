# Implementation Plan: Goals Creation and Management UI and API (FA-GOAL-002)

**Branch**: `024-goals-management` | **Date**: 2026-05-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/024-goals-management/spec.md`

## Summary

Add a full goals management feature: a new `/api/goals` REST endpoint (GET / POST / PATCH / DELETE), a `GoalsContext` with optimistic updates, a `/goals` page with active/completed sections and modal-based create/edit, and a progress bar on each goal card. No schema changes — data model is from FA-GOAL-001.

## Technical Context

**Language/Version**: TypeScript 5.x strict — React 19 (frontend) + Node.js / Express 5.x (backend)
**Primary Dependencies**: Drizzle ORM (existing), Zod 4.x (existing), React Router (existing), `useApi` hook + `useToast` hook (existing)
**Storage**: Neon PostgreSQL via existing Drizzle `goals` table (FA-GOAL-001)
**Testing**: Vitest (unit/component), Playwright (E2E — QA writes after implementation)
**Target Platform**: Browser — desktop and mobile (375px+)
**Project Type**: Full-stack web app (Express backend + React frontend)
**Performance Goals**: Goal list renders < 1 s; optimistic updates feel instant
**Constraints**: No schema changes. TypeScript strict. Follow NetWorthContext and assetsRouter patterns exactly.
**Scale/Scope**: Single user, tens of goals — no pagination needed

## Implementation

### Backend: `src/server/routes/goals.ts`

Follow `assetsRouter` pattern exactly.

```ts
import { Router } from "express";
import { eq, and, asc } from "drizzle-orm";
import { z } from "zod";
import { db } from "../../db/index.ts";
import { goals } from "../../db/schema.ts";
import { authenticateToken, type AuthLocals } from "../middleware/authenticateToken.ts";

export const goalsRouter = Router();

const GOAL_TYPES = ["savings_target", "debt_payoff", "net_worth_milestone", "spending_limit"] as const;
const GOAL_STATUSES = ["active", "achieved", "abandoned"] as const;

const createGoalSchema = z.object({
  name: z.string().min(1).max(100),
  type: z.enum(GOAL_TYPES),
  targetAmount: z.number().min(0),
  targetDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).nullable().optional(),
  linkedAccountId: z.string().uuid().nullable().optional(),
  categoryName: z.string().max(100).nullable().optional(),
})
.refine((d) => d.type !== "spending_limit" || d.categoryName != null, {
  message: "categoryName is required for spending_limit goals", path: ["categoryName"],
})
.refine((d) => d.type === "spending_limit" || d.categoryName == null, {
  message: "categoryName must be null for non-spending_limit goals", path: ["categoryName"],
});

const updateGoalSchema = z.object({
  name: z.string().min(1).max(100).optional(),
  type: z.enum(GOAL_TYPES).optional(),
  targetAmount: z.number().min(0).optional(),
  targetDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).nullable().optional(),
  linkedAccountId: z.string().uuid().nullable().optional(),
  categoryName: z.string().max(100).nullable().optional(),
  status: z.enum(GOAL_STATUSES).optional(),
  currentAmount: z.number().nullable().optional(),
})
.refine((d) => Object.keys(d).length > 0, { message: "At least one field required" })
.refine((d) => !d.type || d.type !== "spending_limit" || d.categoryName != null, {
  message: "categoryName is required for spending_limit goals", path: ["categoryName"],
})
.refine((d) => !d.type || d.type === "spending_limit" || d.categoryName == null, {
  message: "categoryName must be null for non-spending_limit goals", path: ["categoryName"],
});

// GET /api/goals
goalsRouter.get("/", authenticateToken, async (_req, res, next) => { ... });

// POST /api/goals → 201 + goal row
goalsRouter.post("/", authenticateToken, async (req, res, next) => { ... });

// PATCH /api/goals/:id → updatedAt: new Date() on every update
goalsRouter.patch("/:id", authenticateToken, async (req, res, next) => { ... });

// DELETE /api/goals/:id → 204
goalsRouter.delete("/:id", authenticateToken, async (req, res, next) => { ... });
```

**Key PATCH handler logic**: build a `Record<string, unknown>` starting with `{ updatedAt: new Date() }`, then conditionally add each present field. Numeric fields (`targetAmount`, `currentAmount`) are converted to `String()` for DB storage. Use `and(eq(goals.id, id), eq(goals.userId, userId))` — missing or cross-user returns 404.

```ts
export interface ApiGoal {

  type:
    | "savings_target"
    | "debt_payoff"
    | "net_worth_milestone"

  targetDate: string | null; // YYYY-MM-DD
  linkedAccountId: string | null;
  categoryName: string | null;
  currentAmount: string | null; // numeric string or null
  status: "active" | "achieved" | "abandoned";
  createdAt: string;
  updatedAt: string;
}
```

### Frontend context: `src/context/GoalsContext.tsx`

Follow `NetWorthContext` exactly:

- `useEffect` with cancellation token fetches `GET /api/goals` on mount, sets `goals` state
- `addGoal`: tempId optimistic insert → POST → replace with server row or rollback
- `updateGoal`: snapshot previous → optimistic apply (numeric fields as `String()`) → PATCH → replace with server row or restore snapshot
- `removeGoal`: snapshot item + index → optimistic filter → DELETE → splice back on failure
- Export `GoalsProvider` and `useGoals` hook

### Frontend page: `src/pages/GoalsPage.tsx`

- Calls `useGoals()` for `goals`, `isLoading`, `addGoal`, `updateGoal`, `removeGoal`
- Calls `useAccount()` for `accounts` (for the linked account select)
- Splits goals: `active = goals.filter(g => g.status === 'active')`, `completed = goals.filter(g => g.status !== 'active')`
- Renders:
  - Header row with "Add goal" button
  - Active goals section → `<GoalCard>` for each, or empty-state prompt if none
  - Completed/Abandoned section → collapsed by default (`useState(false)` toggle), `<GoalCard>` with de-emphasised styling
- `GoalModal` open state: `null | undefined` (add) or `ApiGoal` (edit)

### Frontend component: `src/components/goals/GoalCard.tsx`

Props: `goal: ApiGoal`, `onEdit: (goal: ApiGoal) => void`, `onDelete: (id: string) => void`, `onStatusChange: (id: string, status: string) => void`

Displays:

- Name + type badge (human-readable label from type value)
- Progress bar: `goalPercent(goal)` — see data-model.md for formula
  - Width = `${percent}%` inline style
  - Label: NZD formatted `currentAmount` / `targetAmount` (Intl.NumberFormat NZD)
  - Note: "Progress will update automatically" when `currentAmount === null`
- Target date (formatted) if set
- Status badge for achieved/abandoned
- Edit button → `onEdit(goal)`
- Delete button → `onDelete(goal.id)` (GoalsPage handles confirmation dialog)
- Status change buttons: "Mark achieved" / "Mark abandoned" (only on active goals)

### Frontend modal: `src/components/goals/GoalModal.tsx`

Props: `goal?: ApiGoal` (undefined = add mode), `onClose: () => void`

State: `name`, `type`, `targetAmount`, `targetDate`, `linkedAccountId`, `categoryName`, `currentAmount` (edit only), `isSubmitting`, per-field error strings

Behaviour:

- When `type` changes to `spending_limit` → show `categoryName` input
- When `type` changes away from `spending_limit` → hide input and clear `categoryName` to `""`
- On save: validate → call `addGoal` or `updateGoal` → close on success; stay open + `setIsSubmitting(false)` on failure
- `isSubmitting` disables both action buttons during the async call (identical to AssetModal)
