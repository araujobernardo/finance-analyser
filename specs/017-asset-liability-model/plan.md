# Implementation Plan: FA-NW-001 — Asset and Liability Data Model

**Branch**: `017-asset-liability-model` | **Date**: 2026-05-11 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/017-asset-liability-model/spec.md`

## Summary

Both `assets` and `liabilities` tables already exist in the live database (created in migration 0000). This feature adds the missing columns — `updated_at` to both tables and `linkedAccountId` to liabilities — via a single additive Drizzle migration. TypeScript types are already exported and will update automatically when Drizzle re-infers them. No data loss, no table recreation.

## Technical Context

**Language/Version**: TypeScript 5.9 / Node 24  
**Primary Dependencies**: Drizzle ORM 0.45, drizzle-kit 0.31, postgres 3.4  
**Storage**: PostgreSQL (Supabase) — existing tables `assets` and `liabilities`  
**Testing**: Vitest — typecheck via `npm run typecheck`  
**Target Platform**: Supabase PostgreSQL (production) + local `.env` DATABASE_URL  
**Project Type**: Web service (schema/migration layer only)  
**Performance Goals**: Migration must complete in under 30 seconds on a live database  
**Constraints**: Additive only — no destructive changes, no data loss for existing rows  
**Scale/Scope**: Two ALTER TABLE statements; 3 new columns total

## Phases

### Phase 1 — Schema changes (T001)

Update `src/db/schema.ts`:

**assets table** — add `updatedAt`:

```typescript
updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
```

**liabilities table** — add `linkedAccountId` and `updatedAt`:

```typescript
linkedAccountId: uuid("linked_account_id").references(() => accounts.id, {
  onDelete: "set null",
}),
updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
```

TypeScript types (`Asset`, `NewAsset`, `Liability`, `NewLiability`) require no changes — Drizzle infers them automatically.

### Phase 2 — Generate migration (T002)

```bash
npm run db:generate
```

Drizzle compares current schema against the last migration snapshot and generates `src/db/migrations/0003_*.sql` containing:

```sql
ALTER TABLE "assets" ADD COLUMN "updated_at" timestamp with time zone DEFAULT now() NOT NULL;
ALTER TABLE "liabilities" ADD COLUMN "linked_account_id" uuid;
ALTER TABLE "liabilities" ADD COLUMN "updated_at" timestamp with time zone DEFAULT now() NOT NULL;
ALTER TABLE "liabilities" ADD CONSTRAINT "liabilities_linked_account_id_accounts_id_fk"
  FOREIGN KEY ("linked_account_id") REFERENCES "public"."accounts"("id") ON DELETE set null;
```

### Phase 3 — Apply migration (T003)

```bash
npm run db:migrate
```

Applies 0003 to the live Supabase database. Existing rows in `assets` and `liabilities` receive `updated_at = now()` as the default; `linked_account_id` on liabilities is null for all existing rows.

### Phase 4 — Verify (T004)

1. `npm run typecheck` — must exit 0
2. `npm run test` — all existing tests must pass (no new tests required; this is schema-only)
3. Confirm columns exist via Drizzle Studio or psql (see quickstart.md Step 2)
4. Confirm existing accounts and transactions are unaffected by signing in to the running app
