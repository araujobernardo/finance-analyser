# Implementation Plan: Auto-Sync Bank Account Balances into Net Worth

**Branch**: `022-auto-sync-bank-balances` | **Date**: 2026-05-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/022-auto-sync-bank-balances/spec.md`

## Summary

Add an `autoSync` flag to `assets` and `liabilities` that, when enabled, automatically sets the record's value to the computed balance of the linked bank account whenever transactions change. Manual value edits suspend auto-sync; a "Re-enable" action restores it. Sync happens synchronously within each transaction mutation request — no background jobs.

## Implementation Phases

### Phase 0: Research ✅ COMPLETE

See [research.md](research.md) for all decisions. Key resolved items:

- Balance computation: single SQL SUM aggregate, `parseFloat`, return 0 for null.
- `balanceClamped` stored in DB (not derived at read time) to avoid per-request re-aggregation on GET.
- Sync runs inline in each transaction mutation handler after `.returning()`.
- PATCH semantics: `value` field present → implicit `autoSync=false`; `{ autoSync: true }` → re-enable + immediate sync.
- Transaction PATCH/DELETE: `accountId` recovered from `.returning()` result.

Run via `npx drizzle-kit migrate`.

### `computeAccountBalance` signature

```ts
import { sql } from "drizzle-orm";
import { transactions } from "../../db/schema.ts";

export async function computeAccountBalance(
  accountId: string,
  userId: string,
  db: DrizzleDB,
): Promise<number> {
  const [row] = await db
    .select({ total: sql<string>`COALESCE(SUM(${transactions.amount}), '0')` })
    .from(transactions)
    .where(
      and(
        eq(transactions.accountId, accountId),
        eq(transactions.userId, userId),
      ),
    );
  return parseFloat(row?.total ?? "0");
}
```
