# Implementation Plan: Savings Category — Rename and Visual Treatment

**Branch**: `005-savings-category-treatment` | **Date**: 2026-04-24 | **Spec**: [spec.md](spec.md)

---

## Summary

Rename the "Savings & Transfers" category to "Savings" throughout the codebase and apply a consistent green colour treatment wherever the category appears. Existing localStorage data containing the old name is normalised on load via a read-time alias — no destructive migration. A new CSS token `--colour-savings` carries the green value used across all affected components. This is a purely client-side, UI + string-replacement change; no new storage keys, no backend.

## Data Model Change

No new fields. No new localStorage keys. The only data concern is backward compatibility:

### Read-time normalisation

When loading transactions from `pfa-v3-transactions`, any transaction whose `category` equals the legacy string `"Savings & Transfers"` must be mapped to `"Savings"` in memory. This map is applied once on load and does not write back to storage until the next user-triggered save (which will use the updated string naturally).

**Implementation location**: `App.tsx`, inside the `useState` initialiser that reads from localStorage — the same place existing transactions are hydrated.
