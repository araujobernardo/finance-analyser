# Implementation Plan: PostgreSQL Database Provisioning and Schema Migrations

**Branch**: `008-postgresql-schema-migrations` | **Date**: 2026-05-05 | **Spec**: [spec.md](spec.md)

---

## Summary

Provision a Supabase PostgreSQL database and introduce Drizzle ORM as the migration system. This creates the foundational data layer for the Finance Analyser's transition from a localStorage-only browser app to a hosted multi-user web application. The deliverable is a committed schema definition, a generated SQL migration, and a one-command migration runner — no API or auth layer.

---

## Technical Context

**Language/Version**: TypeScript 5.9 (strict mode)
**Primary Dependencies**: drizzle-orm, drizzle-kit, postgres (postgres.js), tsx
**Storage**: Supabase PostgreSQL (free tier) — direct connection (port 5432) via `DATABASE_URL`
**Testing**: Vitest (existing) + TypeScript compiler check (`tsc --noEmit`)
**Target Platform**: Node.js (migration scripts) + future Express API server
**Project Type**: Library/infrastructure — no UI, no API in this feature
**Performance Goals**: Migration completes in under 30 seconds on a fresh database
**Constraints**: `src/db/` must not be imported by React components or Vite browser bundle
**Scale/Scope**: Single Supabase project; ~8 tables; schema foundations for multi-user app

## Implementation Tasks

### Task 1 — Install dependencies

```bash
npm install drizzle-orm postgres
npm install -D drizzle-kit tsx
```

New runtime deps: `drizzle-orm`, `postgres`
New dev deps: `drizzle-kit`, `tsx`

No existing dependencies are removed or changed.

---

### Task 2 — `drizzle.config.ts` (project root)

```typescript
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  schema: "./src/db/schema.ts",
  out: "./src/db/migrations",
  dialect: "postgresql",
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
});
```

Drizzle Kit reads `.env` automatically when `DATABASE_URL` is present.

---

### Task 3 — `src/db/schema.ts`

Define all 7 tables using the Drizzle schema from [data-model.md](data-model.md).

Export both table objects (for queries) and inferred TypeScript types (`$inferSelect` / `$inferInsert`) for each entity.

Full schema: see [data-model.md](data-model.md) for column definitions, types, and FK relationships.

---

### Task 4 — `src/db/index.ts`

```typescript
// Server-side only — do not import from React components or Vite browser code.
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";
import * as schema from "./schema";

const client = postgres(process.env.DATABASE_URL!);
export const db = drizzle(client, { schema });
```

Exports a single `db` instance for use by the future API layer.

---

### Task 5 — `src/db/migrate.ts`

```typescript
import { drizzle } from "drizzle-orm/postgres-js";
import { migrate } from "drizzle-orm/postgres-js/migrator";
import postgres from "postgres";

const client = postgres(process.env.DATABASE_URL!, { max: 1 });
const db = drizzle(client);

console.log("Applying migrations...");
await migrate(db, { migrationsFolder: "./src/db/migrations" });
console.log("Done.");
await client.end();
```

Run via `npm run db:migrate` (uses `tsx --env-file=.env`).

---

### Task 6 — Add npm scripts to `package.json`

```json
"db:generate": "drizzle-kit generate",
"db:migrate": "tsx --env-file=.env src/db/migrate.ts",
"db:studio": "drizzle-kit studio"
```

---

### Task 7 — Generate the initial migration

```bash
npm run db:generate
```

This diffs `src/db/schema.ts` against an empty database and writes:

- `src/db/migrations/0000_initial_schema.sql`
- `src/db/migrations/meta/_journal.json`
- `src/db/migrations/meta/0000_snapshot.json`

All generated files are committed to git. The SQL file must be reviewed before committing.

---

### Task 8 — Update `.env.example`

Add or update:

```
DATABASE_URL=postgresql://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres
```

## Testing

### Automated (CI)

- `tsc --noEmit` — verifies schema types compile cleanly with no `any` leakage
- `npx drizzle-kit check` — verifies no schema drift between `schema.ts` and the generated migration snapshot

### Manual (QA gate)

1. Reviewer sets `DATABASE_URL` in `.env` pointing to a Supabase project
2. Runs `npm run db:migrate`
3. Confirms all 7 tables and `__drizzle_migrations` exist in Supabase Table Editor
4. Runs `npm run db:migrate` a second time — confirms idempotent (no error, no duplicate changes)
5. Runs `tsc --noEmit` — confirms zero TypeScript errors
